## Overview

RTX Path Tracing uses a scene file system based on glTF 2.0 with custom extensions. Scenes are defined in `.scene.json` files that reference 3D models, textures, lights, and cameras.

## Loading Scenes

### From Command Line

Load a specific scene at startup:

```bash
Rtxpt.exe --scene bistro.scene.json
```

### From UI

Scenes can be switched at runtime using the **Scene** dropdown in the UI:

1. Launch the application
2. Click the **Scene** dropdown at the top
3. Select a different scene
4. The scene will reload automatically

### Programmatic Loading

From the source code (`Sample.cpp:404`):

```cpp
// Get all scenes in "assets" folder
const std::string mediaExt = ".scene.json";
for (const auto& file : std::filesystem::directory_iterator(GetLocalPath(c_AssetsFolder)))
{
    if (!file.is_regular_file()) continue;
    std::string fileName = file.path().filename().string();
    std::string longExt = (fileName.size()<=mediaExt.length())?(""):(fileName.substr(fileName.length()-mediaExt.length()));
    if ( longExt == mediaExt )
        m_sceneFilesAvailable.push_back( file.path().filename().string() );
}
```

## Scene File Format

### Basic Structure

A `.scene.json` file is based on glTF 2.0 format with custom extensions:

```json
{
  "Graph": [
    {
      "name": "Scene Root",
      "children": [
        // Meshes, cameras, lights go here
      ]
    }
  ],
  "Settings": {
    // Optional: scene-specific settings
  }
}
```

## Asset Pipeline

### glTF 2.0 Support

The scene system is based on glTF 2.0, providing:

- **Industry standard format** - Compatible with Blender, Maya, 3ds Max
- **Physically-based materials** - PBR workflow
- **Animation support** - Node and skeletal animations
- **Extensibility** - Custom extensions for advanced features

From the README:
> "Simple asset pipeline based on glTF 2.0 (support for a subset of glTF extensions including animation)"

### Texture Formats

Supported texture formats:

- **PNG** - Lossless, supports alpha
- **JPEG** - Compressed, no alpha
- **DDS** - Compressed, GPU-native formats (BC1-BC7)
- **EXR** - High dynamic range
- **HDR** - High dynamic range

<Warning>
**Uncompressed Textures**: If the scene contains uncompressed textures, a warning will appear in the UI with an option to batch compress them using `nvtt_export.exe`.
</Warning>

### Texture Compression

For optimal performance, compress textures to DDS format:

1. Launch the application
2. Load a scene
3. If uncompressed textures are detected, the UI will show:
   - "Scene has X uncompressed textures"
   - **Button**: "Batch compress with nvtt_export.exe"
4. Click the button to compress all textures
5. Reload the scene

From `SampleUI.cpp:348`:
```cpp
if (uncompressedTextureCount > 0)
{
    ImGui::TextColored(warnColor, "Scene has %d uncompressed textures", uncompressedTextureCount);
    if (ImGui::Button("Batch compress with nvtt_export.exe", { -1, 0 }))
        if (CompressTextures(m_app.GetUncompressedTextures()))
        {   // reload scene
            m_app.SetCurrentScene(m_app.GetCurrentSceneName(), true);
        }
}
```

## Cameras

### Camera Node Format

Define cameras in the scene graph:

```json
{
  "name": "Cameras",
  "children": [
    {
      "name": "Default",
      "type": "PerspectiveCameraEx",
      "translation": [0.0, 1.8, 5.0],
      "rotation": [0.0, 0.0, 0.0, 1.0],
      "verticalFov": 1.047,
      "zNear": 0.001,
      "enableAutoExposure": true,
      "exposureCompensation": 2.0,
      "exposureValue": 0.0
    }
  ]
}
```

### Camera Properties

- `translation` - [x, y, z] world position
- `rotation` - [x, y, z, w] quaternion
- `verticalFov` - Vertical field of view in radians
- `zNear` - Near clipping plane
- `enableAutoExposure` - Enable auto exposure (boolean)
- `exposureCompensation` - Exposure bias
- `exposureValue` - Manual exposure value
- `exposureValueMin` - Min auto exposure value
- `exposureValueMax` - Max auto exposure value

### Saving Camera Position

### Supported Light Types

From README features:

**Available light types:**

1. **Point Lights** - Omnidirectional point sources
2. **Spot Lights** - Directional cone lights
3. **Directional Lights** - Infinite distance sun lights
4. **Emissive Meshes** - Triangle-based area lights

### Environment Maps

Environment maps provide image-based lighting:

**Supported formats:**
- `.exr` - High dynamic range
- `.hdr` - High dynamic range
- `.dds` - Compressed HDR

Environment maps are auto-discovered from `Assets/envmaps/` folder:

```cpp
// From Sample.cpp:154
m_envMapMediaFolder = GetLocalPath(c_AssetsFolder) / c_EnvMapSubFolder;
```

## Animations

### Enabling Animations

Animations can be enabled:

1. **Via UI**: Check "Enable animations" (requires Realtime Mode)
2. **Via Scene Settings**: Set in `.scene.json`

```json
{
  "Settings": {
    "enableAnimations": true,
    "realtimeMode": true
  }
}
```

### Animation Types

- **Skeletal animations** - Skinned mesh deformation
- **Node animations** - Transform animations
- **Dynamic lighting** - Animated lights

<Note>
Animations are only available in Realtime Mode. They are disabled in Reference Mode for deterministic rendering.
</Note>

## Scene Settings

### Optional Settings Node

Configure scene-specific defaults in `.scene.json`:

```json
{
  "Settings": {
    "realtimeMode": true,
    "enableAnimations": false,
    "enableReSTIRDI": false,
    "enableReSTIRGI": false,
    "startingCamera": 0,
    "maxBounces": 8,
    "realtimeMaxDiffuseBounces": 3,
    "referenceMaxDiffuseBounces": 8,
    "textureMIPBias": 0.0,
    "realtimeFireflyFilter": 0.001
  }
}
```

### Available Settings

| Setting | Type | Description |
|---------|------|-------------|
| `realtimeMode` | boolean | Start in realtime vs reference mode |
| `enableAnimations` | boolean | Enable animations on load |
| `enableReSTIRDI` | boolean | Enable ReSTIR DI (direct illumination) |
| `enableReSTIRGI` | boolean | Enable ReSTIR GI (global illumination) |
| `startingCamera` | integer | Camera index to use (0-based) |
| `maxBounces` | integer | Maximum path bounces |
| `realtimeMaxDiffuseBounces` | integer | Max diffuse bounces in realtime |
| `referenceMaxDiffuseBounces` | integer | Max diffuse bounces in reference |
| `textureMIPBias` | float | Texture LOD bias |
| `realtimeFireflyFilter` | float | Firefly filter threshold |

## Performance Optimization

### Acceleration Structures

The scene uses ray tracing acceleration structures:

- **BLAS** (Bottom-Level) - Per-mesh geometry
- **TLAS** (Top-Level) - Scene instance hierarchy

<Note>
Acceleration structures are automatically rebuilt when the scene changes or when toggling scene elements.
</Note>

### Opacity Micromaps (OMM)

From README:
> "[OMM](https://github.com/NVIDIA-RTX/OMM) integration for fast ray traced alpha testing"

OMMs accelerate rendering of alpha-tested geometry (foliage, fences, etc.).
