## Overview

RTX Path Tracing provides a comprehensive UI for controlling rendering settings, scene management, and camera movement. The interface is built with Dear ImGui and provides real-time feedback.

<Note>
Press **F2** to toggle the UI visibility on/off.
</Note>

## UI Layout

The main UI window appears on the left side of the screen:

- **Primary Window**: Settings panel (left side)
- **Top Controls**: Scene selection and quick settings (top-center)
- **Keyboard Shortcut**: F2 to show/hide UI

From the README:
> "Once the application is running, most of the SDK features can be accessed via the UI window on the left hand side and drop-down controls in the top-center."

## Camera Controls

### Keyboard Movement

- **W** - Move forward
- **S** - Move backward  
- **A** - Move left
- **D** - Move right

### Mouse Controls

- **Left Mouse Button + Drag** - Rotate camera
- **Right Mouse Button** - Pick pixel for debugging
- **Mouse Scroll** - Adjust camera movement speed

### Camera Settings (UI)

Found under **Camera** section:

```
Camera
├── Motion (dropdown)
│   ├── Free flight
│   └── Scene cam 0, 1, 2...
├── Camera position
│   ├── Save - Save to campos.txt
│   └── Load - Load from campos.txt  
├── Aperture (0.0 - 1.0)
├── Focal Distance
├── Keyboard move speed (0.1 - 10.0)
```

### Camera Motion Modes

**Free Flight** (default):
- WASD keyboard controls
- Mouse drag to rotate
- Full freedom of movement

**Scene Cameras**:
- Pre-defined camera positions from `.scene.json`
- Animated camera paths
- Can have custom exposure settings

## Scene Management

### Scene Selection

**Scene** dropdown (top of UI):
- Lists all `.scene.json` files in `Assets/` folder
- Click to switch scenes
- Scene reloads automatically

From `SampleUI.cpp:329`:
```cpp
if (ImGui::BeginCombo("Scene", currentScene.c_str()))
{
    const std::vector<std::string>& scenes = m_app.GetAvailableScenes();
    for (const std::string& scene : scenes)
    {
        bool is_selected = scene == currentScene;
        if (ImGui::Selectable(scene.c_str(), is_selected))
            m_app.SetCurrentScene(scene);
    }
    ImGui::EndCombo();
}
```

## Path Tracer Settings

Under **Path tracer** section (expanded by default):

### Rendering Mode

**Mode** dropdown:
- **Reference** - Accumulate samples for ground truth
- **Realtime** - Real-time path tracing with denoising

From `SampleUI.cpp:610`:
```cpp
int modeIndex = (m_ui.RealtimeMode)?(1):(0);
if (ImGui::Combo("Mode", &modeIndex, "Reference\0Realtime\0\0"))
{
    m_ui.RealtimeMode = (modeIndex!=0);
    m_ui.ResetAccumulation = true;
}
```

### Setup (Reference Mode)

- **Reset** - Reset accumulation buffer
- **Sample count** (1 - 4,194,304) - Target samples per pixel
- **Accumulated samples** - Current progress
- **Pre-warm real-time caches** - Pre-fill temporal buffers
- **Jitter anti-aliasing** - Pixel jitter for AA

### Setup (Realtime Mode)

- **Reset** - Reset temporal caches
- **Samples per pixel** (1 - 64) - SPP per frame
  - Note: Incompatible with ReSTIR DI & GI

### Common Settings

- **Max bounces** (0 - MAX_BOUNCE_COUNT) - Total path length
- **Max diffuse bounces** - Separate diffuse limit
  - Realtime mode: typically 3
  - Reference mode: typically 8
- **Texture MIP bias** - LOD bias for all textures
- **Diffuse sample envmap MIP level** (0-16) - Sky sampling LOD
- **Use Russian Roulette early out** - Stochastic path termination

From `SampleUI.cpp:654`:
```cpp
RESET_ON_CHANGE(ImGui::InputInt("Max bounces", &m_ui.BounceCount));
m_ui.BounceCount = dm::clamp(m_ui.BounceCount, 0, MAX_BOUNCE_COUNT);
if (m_ui.RealtimeMode)
    RESET_ON_CHANGE(ImGui::InputInt("Max diffuse bounces (realtime)", &m_ui.RealtimeDiffuseBounceCount));
else
    RESET_ON_CHANGE(ImGui::InputInt("Max diffuse bounces (reference)", &m_ui.ReferenceDiffuseBounceCount));
```

### Firefly Filter

**Realtime Mode:**
- **FireflyFilter (realtime)** - Enable smart firefly clamping
- **FF Threshold** (0.00001 - 1000.0) - Clamp threshold
- **RX** - Relax on non-noisy paths (higher clamp for direct light)

**Reference Mode:**
- **FireflyFilter (reference *)** - Enable for reference mode
  - Note: Makes results non-deterministic with auto-exposure
- **FF Threshold** (0.01 - 1000.0)

### Post Processing

**Realtime Mode:**

**AA/SR/Denoising** dropdown:
- **Disabled** - No anti-aliasing
- **TAA** - Temporal anti-aliasing
- **DLSS** - NVIDIA Deep Learning Super Sampling
- **DLSS-RR** - DLSS + Ray Reconstruction

**DLSS Quality** (when DLSS/DLSS-RR selected):
- Ultra Performance
- Performance
- Balanced
- Quality
- DLAA (Deep Learning Anti-Aliasing)

From `SampleUI.cpp:720`:
```cpp
const char* items[] = { "Disabled", "TAA", "DLSS", "DLSS-RR" };
if (ImGui::BeginCombo("AA/SR/Denoising", items[m_ui.RealtimeAA]))
{
    // ... selection logic
}
```

**Standalone Denoiser:**
- **Use standalone denoiser (NRD)** - Enable NVIDIA Real-Time Denoisers
  - Disabled when DLSS-RR is active

**Reference Mode:**

- **Photo mode screenshot** - Denoise with OptiX/OIDN
  - Saves `photo.bmp` with denoised output
  - Uses external command-line denoiser

**Common:**
- **Enable tone mapping** - Toggle tone mapping pass

### Light Sampling

**Realtime Mode:**

- **Use ReSTIR DI (RTXDI)** - ReSTIR direct illumination
  - Requires NEE enabled
  - Incompatible with DLSS-RR (by default)
- **Use ReSTIR GI (RTXDI)** - ReSTIR global illumination
  - Incompatible with DLSS-RR (by default)

**Common:**

- **Use Next Event Estimation** - Direct light sampling (NEE)

**NEE Settings:**
- **Sampling technique** dropdown:
  - Uniform
  - Power+
  - NEE-AT (Adaptive Temporal)
- **Candidate samples** (1-16) - Candidates per full sample
- **Full samples** (0-63) - Shadow-tested samples
- **Boost samples on dominant surface** (0-16) - Extra samples on primary surface

From `SampleUI.cpp:820`:
```cpp
RESET_ON_CHANGE(ImGui::InputInt("Candidate samples", &m_ui.NEECandidateSamples, 1));
m_ui.NEECandidateSamples = dm::clamp(m_ui.NEECandidateSamples, 1, 16);
RESET_ON_CHANGE(ImGui::InputInt("Full samples", &m_ui.NEEFullSamples, 1));
m_ui.NEEFullSamples = dm::clamp(m_ui.NEEFullSamples, 0, 63);
```

### Performance

**Shader Execution Reordering:**

- **NVAPI HitObject codepath** - Use NVAPI extensions
  - **NVAPI ReorderThreads** - Enable SER
- **dx::HitObject codepath** - Use DirectX native extensions (Agility SDK 619+)
  - **dx::MaybeReorderThreads** - Enable SER

### Debugging

- **Realtime noise** - Enable per-frame noise variation
  - When disabled, noise seed is constant (useful for A/B testing)

## Light Pre-processing

Under **Light pre-processing** section:

### Info and Statistics

- Light count
- Emissive triangle count
- Importance sampling statistics

### Distant Lighting

- Environment map importance sampling settings
- Directional light configuration

### NEE-AT Settings

**Temporal Feedback:**
- **Global temporal feedback** - Scene-wide light importance
  - **Global feedback ratio** (0.0 - 0.95)
- **Local temporal feedback** - Per-tile light importance
  - **Local feedback ratio** (0.0 - 0.95)
- **BSDF vs NEE-AT MIS boost** (0.0 - 1000.0) - MIS weight tweak
- **Distant vs Local initial importance** (0.01 - 100.0) - Balance distant/local lights
- **Anti-lag pre-pass (experimental)** - Same-frame guidance

## Performance Monitoring

The UI displays:

- **Frame time** - Milliseconds per frame
- **FPS** - Frames per second
- **Resolution** - Render → Display resolution
- **Scene stats** - Lights, materials, meshes, instances

<Note>
With DLSS-G (frame generation), the FPS counter shows effective FPS with an asterisk and notes the multiplier.
</Note>

From `Sample.cpp:842`:
```cpp
if (m_ui.DLSSGMultiplier != 1)
    m_fpsInfo = StringFormat("%.3f ms/%d-frames* (%.1f FPS*) *DLSS-G", 
        frameTime * 1e3, m_ui.DLSSGMultiplier, m_ui.DLSSGMultiplier / frameTime);
else
    m_fpsInfo = StringFormat("%.3f ms/frame (%.1f FPS)", frameTime * 1e3, 1.0 / frameTime);
```
