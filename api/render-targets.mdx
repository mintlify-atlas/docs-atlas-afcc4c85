## Overview

The `RenderTargets` class manages all GPU textures and buffers used throughout the RTX path tracing pipeline. It handles creation, resizing, and clearing of render targets including radiance buffers, denoising buffers, motion vectors, depth, and post-processing targets.

void RenderTargets::Init(
    nvrhi::IDevice* device,
    donut::math::uint2 renderSize, 
    donut::math::uint2 displaySize,
    bool enableMotionVectors,
    bool useReverseProjection,
    int backbufferCount)
{
    m_UseReverseProjection = useReverseProjection;
    m_BackbufferCount = backbufferCount;
    m_device = device;
    m_renderSize = renderSize;
    m_displaySize = displaySize;
    
    nvrhi::TextureDesc desc;
    desc.width = renderSize.x;
    desc.height = renderSize.y;
    desc.sampleCount = 1;
    desc.dimension = nvrhi::TextureDimension::Texture2D;
    desc.keepInitialState = true;
    desc.mipLevels = 1;
    
    // Create depth buffer
    desc.format = nvrhi::Format::R32_FLOAT;
    desc.isUAV = true;
    desc.initialState = nvrhi::ResourceStates::UnorderedAccess;
    desc.debugName = "Depth";
    Depth = device->createTexture(desc);
    
    // Create motion vectors
    desc.format = nvrhi::Format::RGBA16_FLOAT;
    desc.debugName = "ScreenMotionVectors";
    ScreenMotionVectors = device->createTexture(desc);
    
    // ... create all other render targets
}

## Render Target Types

### Primary Radiance Buffers

<ParamField path="AccumulatedRadiance" type="nvrhi::TextureHandle">
  **Format:** `RGBA32_FLOAT`
  
  **Usage:** Accumulates path-traced radiance in non-realtime (reference) mode. Each frame adds to this buffer for progressive refinement.
</ParamField>

<ParamField path="OutputColor" type="nvrhi::TextureHandle">
  **Format:** `RGBA16_FLOAT`
  
  **Usage:** Raw path tracing output for the current frame. This is the input to TAA/DLSS in realtime mode.
</ParamField>

<ParamField path="ProcessedOutputColor" type="nvrhi::TextureHandle">
  **Format:** `RGBA16_FLOAT`
  
  **Usage:** Post-processed color after TAA/DLSS upscaling, but before tone mapping. Full display resolution.
</ParamField>

<ParamField path="LdrColor" type="nvrhi::TextureHandle">
  **Format:** `SRGBA8_UNORM`
  
  **Usage:** Final tone-mapped LDR color ready for display.
</ParamField>

<ParamField path="LdrColorScratch" type="nvrhi::TextureHandle">
  **Format:** `SRGBA8_UNORM`
  
  **Usage:** Scratch buffer for ping-ponging post-process effects with LdrColor.
</ParamField>

### Geometry Buffers

<ParamField path="Depth" type="nvrhi::TextureHandle">
  **Format:** `R32_FLOAT`
  
  **Usage:** Screen-space depth buffer. Exported by path tracer, used by TAA and other post-processes. Supports reverse-Z projection.
</ParamField>

<ParamField path="ScreenMotionVectors" type="nvrhi::TextureHandle">
  **Format:** `RGBA16_FLOAT`
  
  **Usage:** Screen-space motion vectors for temporal reprojection. Used by RTXDI, TAA, and temporal denoising.
</ParamField>

<ParamField path="Throughput" type="nvrhi::TextureHandle">
  **Format:** `R32_UINT`
  
  **Usage:** Path throughput after perfect specular reflections. Used by RTXDI for correct MIS weighting with PSR (Perfect Specular Reflections).
</ParamField>

<ParamField path="SurfaceDataBuffer" type="nvrhi::BufferHandle">
  **Stride:** `sizeof(PackedPathTracerSurfaceData)`
  
  **Usage:** Packed G-buffer containing surface properties (position, normal, material) with temporal history.
</ParamField>

## Usage Example

```cpp
// Create and initialize render targets
RenderTargets renderTargets;
renderTargets.Init(
    device,
    uint2(1920, 1080),  // Native render size
    uint2(3840, 2160),  // Display size (DLSS upscaled)
    true,               // Enable motion vectors
    true,               // Use reverse-Z projection
    3                   // Triple buffering
);

// Check if resize needed
if (renderTargets.IsUpdateRequired(newRenderSize, newDisplaySize))
{
    // Recreate render targets at new resolution
    renderTargets.Init(device, newRenderSize, newDisplaySize, ...);
}

// Clear buffers at frame start
commandList->open();
renderTargets.Clear(commandList);

// Use render targets in path tracing
bindingSet->bindTexture(0, renderTargets.OutputColor);
bindingSet->bindTexture(1, renderTargets.Depth);
bindingSet->bindTexture(2, renderTargets.ScreenMotionVectors);

// Access denoising outputs
for (int i = 0; i < cStablePlaneCount; ++i)
{
    auto diffuse = renderTargets.DenoiserOutDiffRadianceHitDist[i];
    auto specular = renderTargets.DenoiserOutSpecRadianceHitDist[i];
}
```

## Notes

- All render targets use `keepInitialState = true` to avoid unnecessary state transitions
- The class supports both native rendering resolution and display resolution for DLSS/upscaling workflows
- Reverse-Z projection improves depth precision for large scenes
- Stable planes enable multi-resolution path tracing for better denoising quality
